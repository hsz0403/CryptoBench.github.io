论文摘要

为什么需要一个加密原生的 Agent Benchmark？

一般 Agent 指标忽视了加密市场的高时效性、对抗性与多源数据需求。CryptoBench 通过四象限任务体系（简单/复杂 × 检索/预测）和每月更新的题库，检验模型在真实 analyst 工作流中的检索、综合与推理能力。实验揭示了 “retrieval-prediction imbalance”——许多顶尖模型在查找数据时表现优秀，但在预测和综合环节明显失衡。

场景驱动

题目由 DeFi 风控、量化交易员、链上调查员共同撰写，严格对应真实工作流。

动态更新

通过模板化变量（地址、Ticker、时间窗）实现季度刷新，降低模型被“背答案”的可能。

工具编排

任务要求访问官方 Dashboard、DEX/衍生品终端、链上情报和 AI signal 平台。

Agent 友好

同一题目同时评估原生 LLM 与 SmolAgent 流程，检验规划与工具调用对成绩的影响。

论文贡献

CryptoBench 的三项核心贡献。

任务专业性与多样性

覆盖 on-chain intelligence、市场数据、DeFi/Oracle、DEX & Derivatives、MEV & AI Signals 五大纵深领域。

动态“活”基准

季度更新题库，引入新叙事（如 restaking、模组化链、intent layer），保持长期有效性。

广泛平台覆盖

迫使 Agent 选择正确的数据源、处理 UI 差异并综合冲突信息，真实模拟分析师日常。

方法概览

从任务设计到评分的端到端流水线。

下列流程对应 main.tex 的 Section 3（Benchmark）与 Section 4（Experiments）。

任务模板

根据当前市场结构实例化变量（地址、交易哈希、时间窗口）。

参考答案

作者从权威 Dashboard 与交易终端抓取数据并截图存档。

模型设置

评估 Grok-4 系列、GPT-5、Qwen3-Max、Gemini 2.5 Pro 等 10 个模型。

Agent 执行

分别运行直接 LLM 调用与 SmolAgent 版本，保持工具权限一致。

评分与分析

按正确性、综合度、预测准确率和引用来源打分，并记录失败模式。

质量控制

Dataset Construction & Quality Control

根据论文 Section 3.2，题目由加密原生团队双人复核：一人设计、一人验证答案，必要时回溯链上记录。

  1. Prompt 模板与变量锁定

    把 token、钱包、交易对、时间窗拆分为可替换变量，降低手工成本。

  2. 证据采集

    为每题附上来源链接与截图，提供审计轨迹。

  3. 交叉审阅

    第二位作者独立复现操作，确认可重复性。

  4. 上线与监控

    题目进入当月发布池，异常反馈会触发修订与变量切换。

任务象限

Retrieval × Prediction 的四象限刻画。

Figure1.pdf 与 Figure2.pdf 展示了象限和整体流程，下方 PDF 画廊可直接预览。

01 · Simple Retrieval

高速查询最新的 TVL、Funding Rate、Whale Flow。大多数模型得分最高。

02 · Complex Retrieval

多跳搜索 + 多源比对，惩罚浅层搜索与旧数据。

03 · Simple Prediction

以实时数据为支撑的短期预测，GPT-5 在此仅 6.25% 准确率。

04 · Complex Prediction

需综合多平台证据做情景推演，是区分 Agent 水平的关键。
